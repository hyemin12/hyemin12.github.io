### 깃에서 자주 쓰이는 명령어

#### ■ `git add`

작업 디렉토리의 변경 내용을 스테이징 영역에 넘길 때 사용하는 명령어

- `git add 파일명/디렉토리경로`: 변경 내용의 특정 파일이나 디렉토리 경로만 스테이징 영역에 넘길 때

- `git add .`: 변경 내용의 전체 파일을 스테이징 영역에 넘길 때

#### ■ `git status`

작업 디렉토리와 스테이징 영역의 **상태를 확인**하기 위해서 사용

#### ■ `git commit`

`git add`로 스테이징 영역에 추가한 변경 내용들을 버전을 만드는 명령어

1.`git commit`을 터미널에 입력

![git commit](https://github.com/hyemin12/react-dashboard/assets/66300732/ecfcbf33-8944-4b1b-9ab5-c0c44b7e4fa6)

2.위와 같은 메세지가 뜨면, 내용을 변경한다는 의미로 `CC`를 입력

![commitmessage](https://github.com/hyemin12/react-dashboard/assets/66300732/72768c08-055c-4d62-9a7d-9730f0f1348d)

3.`---INSERT---`가 뜬다면 첫번째 줄에 메세지 입력

![commit나오기](https://github.com/hyemin12/react-dashboard/assets/66300732/52382d93-34b5-4db4-9282-0d85dfe8bb0f)

4.`ESC`키를 누른 다음 `:wq` 입력하고 편집기에서 나오기

![commit 결과](https://github.com/hyemin12/react-dashboard/assets/66300732/f44dce04-92e8-474f-8a29-f458723aeead)

5.위와 같은 메세지가 뜬다면 성공

#### ■ `git log`

commit 내역을 확인하는 명령어

```bash
commit 0308531bd1d56b01a3db0f543bbe9e7cf9e6b7d8 (HEAD -> main)
Author: hyemin12 <-@naver.com>
Date:   Thu Nov 16 10:05:47 2023 +0900

    11/16 공부내용
```

#### ■ `git remote`

연결 저장소와 관련된 명령어

- `git remote -v`: 연결 저장소 확인
- `git remote 원격저장소별칭 원격저장소URL`: 연결 저장소와 연결하기

### VSCODE에서 GUI로 init, add, commit 하기

VSCODE 사이드바의 3번째 아이콘(Source Control)을 사용하면 CLI 보다 편리하게 git을 관리할 수 있다.

![gui Init](https://github.com/hyemin12/react-dashboard/assets/66300732/b7c62fd9-e5ee-4e5a-b850-3ce6fddc774a)

Initialize Repository 버튼은 CLI의 명령어 중 `git init`과 같은 행동을 수행함

![guiadd](https://github.com/hyemin12/react-dashboard/assets/66300732/576cebc5-5777-466f-b895-15241b1d30b8)

Message 박스에 메세지를 입력하고, Commit 버튼을 누르면 CLI 명령어인 `git commit -m '커밋 메세지'와 동일하게 커밋이 수행됨

만약, `git log`와 같이 git의 history가 보고싶다면 VSCODE의 "extensions"에서 git history를 검색하고 설치하면 됨

![extentiongit](https://github.com/hyemin12/react-dashboard/assets/66300732/38810d82-7c2c-4020-ad68-92fc35f3add8)

![githistory](https://github.com/hyemin12/react-dashboard/assets/66300732/9ebb9e3d-b309-413d-b394-fd1aed61f02e)

설치하고 나면 Source Control 메뉴의 상단 부분에 시계모양이 나타나는데 이를 클릭하면 git history를 확인할 수 있다.
